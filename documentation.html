Database Connection

It doesn't require much stress to configure a working database in codefii. The configuration file is found in the root directory of your codefii app as a .env file. Codefii doesn't support multiple database connection like postgres and Sqlite, it only supports Mysql so all you need is add the neccessary settings and you're good to go. 


                                    APP_NAME=Codefii 
                                    APP_ENV=local
                                    APP_URL=http://localhost
                        
                                    DB_CONNECTION=mysql
                                    DB_HOST=127.0.0.1
                                    DB_PORT=3306
                                    DB_DATABASE=codefii
                                    DB_USERNAME=root
                                    DB_PASSWORD=xxxxxx
                                    DB_PREFIX=null

Deployment

Deploying your project is pretty straightforward,the only file that needs to be modified when deploying to shared hosting or vps is the .env which happens to be your database settings and you're good to go.

Understanding the Model and FiiORM

The model is actually the M in MVC which is the layer of the system responsible for representing business logic. The model in codefii follows the Active Record pattern using the Fiirm, fii relational model or fiiorm. The aim is to educate developers on how to write and read from database.

Object Relational Mapping

The object relational mapping often abbreviated as ORM is a technique that connects the rich objects of an application to tables in a relational database management system. Using Fiirm or FiiORM, the properties and relationships of the objects in an application can be easily stored and retrieved from a database without writing SQL statements directly and with less overall database access code even though the Fiirm has made provision for writing SQL statements when neccessary.

Fiirm as a framework

The Fiirm is flexible and super easy, with several mechanisms of which the most important thing is the ability to:

  • Represent models and their data
  • Represent association between these models
  • Perform database operations in object oriented fashion.
  • Use a class name as a table name

Fiirm and Auth

The auth is a special class that extends to the Fiirm, and is capable of handling or performing the following function:

  • Log Users in
  • Check if user sessions exist

Most likely, if you're writing a Model that has to do with just mere CRUD, extending to the Fiirm from your model is advisable. In a case where you want to log a user into your application, the auth class has already save you the stress of writing much code.

Updating Record

The update technique is very easy and straightforward provided you know the id of the record you want to update.The update technique only accept accociative array of which the first key and value are known as the active properties. In our case active properties represents the id or primary key column in your table. From our previous chapters the author name was set to Prince E. Darlington, let's try updating it to use Clinton Nzedima as the new author name. 

         
$books = new Books();
$books->Update(['id'=>1,'author'=>'Clinton Nzedima']);

The SQL intepretation of the above snippet is:

         
"UPDATE books SET author='Clinton Nzedima' WHERE id=1";

Creating And Rendering The View

The MVC (model-view-controller) architecture is designed in such a way that different experts can work on one project at the same time without any kind of complications. The view as the name implies is the physical component of any App of which users relate with quickly. The view directory where pages are stored is located in the /templates directory.

There two ways to use view in codefii, one is by using the defualt view from the controller class. example:

         
class Test extends Controller {
      public function inex(){
      return $this->view(['Pages/hello','Pages/partial']);
     }
}

Another way to use view is by importing the View class from the Core directory

         
use Core\View\View;
class Test extends Controller{
       public function index(){
        View::render('Pages/index');
       }
}

When using View::render(); function, your view files likely to be a .php file likewise $this->view(); otherwise it might not work as expected.

Understanding View Partials

I know most php programmers are familiar with the use of require, ever since the web started growing drastically, the arrangment of files on the web has also change thereby making it easier to manage. These files has been seperated into components for reusability sake. In the codefii, a mere html file is seperated into four parts namely.

  • Partial
  • Header
  • Body
  • Footer

Partial

Partial or partials is the first file read by the server whenever a given url is called. For instance, the meta tag, link tag, title tag and etc. These set of tags are known as the partial tags because they probably remain unchanged on every page, but sometimes only the title keeps changing as page changes; so in Codefii they are treated seperately.

         
class Hello extends Controller {
     public function index(){
      return $this->view(["Pages/index","Pages/partial"]);
    }
}

Header

With no much words, the header in this context is reffered to as Navigation Menu. Every unique website mentains one navigation menu that's why we choose to call it Header in codefii, and it's located in the /header directory. Every Header is pre-configured in the Codefii/Controllers/Controller.php file for dynamic loading i.e

         
namespace Codefii\Controllers;
use Core\Controller\BaseController;
abstract class Controller extends BaseController{
    public function init(){
        $this->setHeader("header/header"); //always create a constructor
  }
}

Understanding the Controller

Action Controller is the C in MVC. After the router has determined which controller to use for a request, the controller is responsible for making sense of the request, and producing the appropriate output. Luckily, Action Controller does most of the groundwork for you and uses smart conventions to make this as straightforward as possible.

A controller can thus be thought of as a middleman between models and views. It makes the model data available to the view so it can display that data to the user, and it saves or updates user data to the model.

Controller Naming

A controller in Codefii is just like the normal class naming in php, but qualifying your class name with Controller allows you to use the default class name in route without attaching controller prefix alongside.

Methods And Actions

Controllers on Codefii are meant to inhrerit from Controller and has method just like any class in php. Methods are executed when the controller recieves a request, then Codefii creates an instance of runs the method called in the route

 
use App\controllers\Controller;
class HomeController extends Controller{
       public function firstPage(){
        echo "this is my first page";
     }
}

If a user goes to Home/firstPage in your application, Codefii will create and instance of HomeController and call its firstPage method.

Query String Parameters

You will probably want to access data sent in by the user or other parameters in your controller through query string parameters which refers to those parameters that are sent as part of the URL. https://codefii.com/users?author=prince+darlington

 
use Codefii\Http\Request;
 Class HomeController extends Controller{
    public function users(){
    echo Request::get('author');
    }
}

Getting Routes Parameter

Route parameter is a set of rules defined for checking valid URL query string using regular expression. You may have a regex defined as {id:([a-zA-z0-9])*} in your routes as:

$router->routes('/prince/{id:([a-zA-z0-9])*}')

which means that if a user goes to /Home/LsldfnfK34 (the characters after the last first slahes after prince is an assumed id), while your controller has:

 
 Class HomeController extends Controller {
    public function hello(){
     echo $this->route_params['id'];
   }
}

The output of the above code would be: LsldfnfK34 or characters you pass into the url.

Routing

The codefii router recognizes URLs and dispatches them to a controller's action or method. It can also generate paths and URLs, avoiding the need to hardcode strings in your views.

Coonecting URLs

$router->routes('/',['YourControllerName@method']);

In Codefii routing, there's no need to attach the prefix Controller to the controller name i.e Note: Never write a controller like this: HomeController@method in your route, but rather, this is advisable Home@method

Routing With Namespace

In a case where you want to change your controllers' default folder to something different, there's always a need to register it in your route thereby telling the rout the location of the class and its method. 

$router->routes('/',['Admin\\Home@index']);

The above code tells the route to use Admin as the namespace, meaning that your new folder structure would look like this: App/Controllers/Admin/.

Routing With Parameters

$router->routes('/posts/{id:([a-zA-Z0-9])}',['Home@index']);
